About My Website

Posted on 2020-03-12 In Record Views:

About This Site

At the time I write this post, this website is running on an Aliyun ECS, using the framework Hexo. Currently (2020-03-12 21:41:23) it hosts only my personal blog in default NexT theme. Support for Math expressions and other further bugs features haven’t been explored. Totally a bare blog.

Q & A

(Self-questioning)

“Who are you?”

See my profile. It may will be updated later.

“Why do you build a website?”

Because some of my friends do, for example rabbit-lb, he has posted (still posting?) many wonderful articles about Math and influences my life a lot. I deeply admire his talent and effort, and want to learn more to become as excellent as he is. (Wish you NEVER see this page?)

Anyway, I want to write down what I’ve learned again and strengthen my knowledge. I want to be what those extraordinary people are by devoting myself into wonderful things they might do. It’s a long way to advance. Hope I remember this tomorrow.

Recently I am pushing to get into the school of CS in my university.

Basically like what you see by searching “Hexo”, “VPS”, “Git” and “Nginx”. It took me a whole afternoon. I will write a new article someday to explain this and note some issues I have met.

博客结构

目前（2020年8月10日15点17分）是这样规划的

Categories

按 Hexo 的逻辑，“类别”这个应该是每篇博客只有一个，再添加就是子类。基本会设置 Record、Reading、Cases 这三类，技术记录、读书笔记和做题记录。

内容上很可能有重叠，比如读书的时候会遇到例题要做。这些就就近写了，以主要内容优先。

Record

记录自己折腾的一些东西，例如搭网站、搞Linux这类。应该会是类目最多的。

Reading

是读书笔记。也会写一些读文档的记录。虽然手头的书都还没读完。应该会全是技术类书籍和教材比如CS:APP、编程珠玑这类的。不太可能有文学类书籍，如果有的话很可能是轻小说？那种东西应该不会被我放上来吧，毕竟我读着就是消遣。尽量放一些经典的小说，还有社会类的书。手头还是积了不少这样的东西的。

写博客的话，算是一个身份不明的人对着一群身份不明的人做广播？把自己的什么东西都往外讲的可能性不小。即使不考虑网络环境，顺着 Github 和邮箱也能找到我的真人。还是注意一点比较好。

一本书/一篇文档的全部笔记不会发到一篇文章里，除非它特短。关于来源，会在该书/文档的第一篇笔记的开头注明ISBN/网址/其他可靠的信息。书名以该书 CIP 为准。

Cases

错题集

记录自己刷题的心碎路历程。目前还是一篇都没有呢。会抽空把之前补过的题都放上来。多校赛好难。

会注明题目来源和题号等信息。

2023 年 1 月 15 日更新：Cases 收录零碎的记录，例如项目中用到的部分算法或设计模式、平时的小型技术问题等等，而 Record 负责“长篇连载”类型的记录。

Tags

Categories 代表一篇博客的形式和来源，而 Tags 负责标注它的具体内容。多个 tag 可以共存。比如搭云盘的博客标上了 Network，搭 Minecraft 服务器的博客则有 Network 和 Game 这两个tag。

应该就这些了。后续再有结构性的调整，也会写在这里。
